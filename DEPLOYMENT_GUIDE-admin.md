# دليل نشر البوابة المركزية (Deployment Guide)

هذا الدليل يشرح كيفية رفع بنية **Central Admin Portal** على سيرفر إنترنت لتمكين الفروع من المزامنة والربط اللحظي.

## 🚀 الخيارات المتاحة للنشر المجاني

للبدء بتكلفة صفر، ننصح بالخدمات التالية التي تدعم Node.js و WebSockets:

### 1. Render.com (الأسهل)
*   **المزايا:** ربط مباشر بـ GitHub ونشر تلقائي.
*   **الخطوات:**
    1. أنشئ حساباً واربطه بـ GitHub.
    2. اختر "New Web Service".
    3. اختر مستودع (Repository) الخاص بالمشروع.
    4. استخدم الأوامر التالية:
        - **Build Command:** `npm install && npx prisma generate`
        - **Start Command:** `node server.js`
    5. في إعدادات البيئة (Environment Variables)، أضف `DATABASE_URL` و `JWT_SECRET`.

### 2. Railway.app (الأسرع في الأداء)
*   **المزايا:** توفر قاعدة بيانات PostgreSQL و Redis في ثوانٍ.
*   **الخطوات:** تشبه Render ولكنها توفر واجهة رسومية لإدارة قواعد البيانات.

---

## 🛠️ خطوات تقنية هامة

### 1. قاعدة البيانات (Prisma & PostgreSQL)
بما أننا نستخدم **SQLite** للتطوير المحلي، يجب تغيير الـ `provider` في ملف `prisma/schema.prisma` إلى `postgresql` عند الانتقال للسيرفر:

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

### 2. إعدادات الفروع (Branch Config)
بعد الحصول على رابط الأدمن (مثلاً: `https://my-admin.onrender.com`):
1. اذهب لكل جهاز في الفروع.
2. افتح ملف الـ `.env`.
3. حدث قيمة `PORTAL_URL` لتشير لهذا الرابط.
4. أعد تشغيل برنامج الفرع.

---

## 🔒 الممارسات الأمنية الموصى بها
- اترك قيمة الـ `PORT` ليتعامل معها السيرفر تلقائياً.
- تأكد من أن الـ `API_KEY` لكل فرع قوية وغير معروفة.
- استخدم **HTTPS** دائماً (الخدمات السابقة توفره مجاناً).

---
**جاهز للنشر!** بمجرد رفع الكود لهذه الخدمات، سيبدأ محرك المزامنة اللحظية بالعمل فوراً.
